# «История двух идиотов»

Готовый статический сайт-подарок на день рождения Никиты, собранный по подготовленному анализу Telegram-переписки Богдана и Никиты за период **05.12.2018–13.07.2026**.

## Концепция

Визуальный язык сочетает секретный архив, терминал, карточки личного дела и интерфейс переписки, но не копирует Telegram. Основная палитра — графит, молочный текст и один кислотный акцент. Начало сайта построено на абсурде, мемах и статистике; ближе к финалу фон и композиция становятся спокойнее, а содержание переходит к конкретным проявлениям поддержки.

В сайте нет внешних шрифтов, библиотек, аналитики, cookies и сетевых запросов. Он работает локально и может быть размещён на любом статическом хостинге.

## Локальный запуск

### Самый простой способ

Откройте `index.html` двойным кликом. Большинство функций будет работать сразу.

### Рекомендуемый способ

Из папки проекта выполните:

```bash
python -m http.server 8000
```

Затем откройте в браузере:

```text
http://localhost:8000
```

Этот способ корректнее для локальной загрузки аудио и изображений.

## Где менять контент

Весь контент вынесен в `data.js` в объект:

```js
window.FRIENDSHIP_DATA = { ... }
```

Основные блоки:

- `siteConfig` — название, именинник, период, подзаголовок и альтернативные названия;
- `dossier`, `stats`, `facts` — вводный блок и статистика;
- `dictionary`, `languageStats` — локальные мемы и языковые особенности;
- `timeline`, `storylines`, `plans`, `debates` — хронология и большие истории;
- `quotes`, `miniStories`, `awards`, `quiz` — развлекательные блоки;
- `responseGenerator` — синтетические ответы с обязательной маркировкой;
- `gallery`, `audio` — пути к медиа;
- `warmMoments`, `finalMessage` — тёплый финал;
- `reviewLists` — редакторский чек-лист.

Каждая контентная запись содержит поля:

```js
{
  title,
  text,
  date,
  author,
  category,
  privacyLevel,
  approvedForPublication,
  sourceReference
}
```

Некоторые типы записей содержат дополнительные поля: `context`, `winner`, `probability`, `quote`, `themes` и другие.

## Публичная и личная версии

По умолчанию открывается **безопасная публичная версия**.

Логика фильтрации:

```js
approvedForPublication: true  // отображается в публичной версии
approvedForPublication: false // скрыто публично, доступно в личной версии
```

Переключатель находится в правой части верхнего меню. Выбранный режим сохраняется в `localStorage` текущего браузера.

Личная версия не показывает материалы, которые были исключены полностью: контакты, адреса, банковские данные, медицинские подробности, рабочие документы, интимные материалы и опасные переписки третьих лиц. Эти сведения не записаны в `data.js` вообще.

7. `assets/audio/voice-nikita-01.mp3` — короткая безопасная характерная реплика Никиты.

9. `assets/audio/final-message.mp3` — личное аудиопоздравление продолжительностью 30–60 секунд.

## Цитаты, требующие ручного подтверждения

- «Почитай мое произведение» — из-за медицинского контекста.
- Диалог про Fortnite и кариес — только закрытая версия.
- «Рисуй везде Калашникова» — только с пояснением про визуальное техническое задание.
- Карьерные цитаты про приукрашивание резюме — не публиковать рядом с профессиональным профилем.
- «Слит шлюха» и другие 18+ формулировки — только личная версия.
- История выбора iPad — только обезличенный пересказ.
- Все фотографии, скриншоты и голосовые — отдельное ручное подтверждение.

## Недостающие данные

В анализе отсутствует подтверждённая частотность эмодзи по участникам. В соответствующем разделе сайта оставлен явный плейсхолдер. Чтобы заполнить его корректно, нужно отдельно посчитать эмодзи из исходного Telegram JSON без стикеров и реакций.

Также пока отсутствуют оригинальные фотографии и безопасные фрагменты голосовых.

## Размещение сайта

### GitHub Pages

Подходит для бесплатной публичной версии. Создайте репозиторий, загрузите содержимое папки и включите Pages для основной ветки.

### Netlify или Cloudflare Pages

Подходит для быстрого размещения через drag-and-drop или Git. Сборка не требуется; publish directory — корень проекта.

### Личная версия без публичной ссылки

Передайте ZIP-архив имениннику или откройте сайт локально на ноутбуке/телефоне. Это наиболее безопасный вариант для материалов `approvedForPublication: false`.

### Важное ограничение

Переключатель «личная версия» — интерфейсная фильтрация, а не криптографическая защита. Если личные данные залиты на публичный хостинг, технически их можно найти в `data.js`. Для реального разделения создайте две копии проекта и удалите личные элементы из публичного `data.js`.

## Перед финальной отправкой

1. Добавьте главный совместный кадр.
2. Проверьте все элементы личной версии вместе с именинником либо оставьте сайт только локальным.
3. Прослушайте каждый аудиофайл на предмет имён, адресов и фоновых разговоров.
4. Проверьте скриншоты на телефоны, логины, названия чатов и третьих лиц.
5. При публичном размещении рекомендуется физически удалить все записи `approvedForPublication: false` из `data.js`.
